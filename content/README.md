# Content Folder — Edit Here in Obsidian

This is where all the website content lives. Edit these files in Obsidian, commit, and the website updates automatically.

---

## 📁 Folder Guide

| Folder | What goes here |
|---|---|
| `meetings/` | One file per meeting plan |
| `activities/` | One file per activity, craft, or badge project |
| `communications/` | One file per parent email, permission slip, or newsletter |
| `resources/` | One file per template, checklist, or reference guide |
| `onboarding/` | The 5 onboarding guides — edit but don't rename these |
| `pages/` | Small bits of static text on each page (tips, etc.) |

---

## ✍️ How to Add Something New

1. Open the right folder (e.g. `meetings/`)
2. Create a new file — name it something like `2026-04-05-spring-hike.md`
3. Paste the template below at the top, fill it in
4. Write your content below the `---` line
5. Save → commit → the site rebuilds automatically

---

## 📋 File Template

Copy this to the top of every new file:

```
---
title: Your Title Here
icon: 📅
tags: [Tag1, Tag2]
description: One or two sentences describing this item. Shows up on the card preview.
---

Your full content goes here. Use regular Markdown formatting.

## Section Heading

- Bullet point
- Another bullet point

**Bold text**, *italic text*, regular text.
```

## 💡 Tips

- **File names** use lowercase letters, numbers, and hyphens only — no spaces or special characters
- **Dates** in filenames help keep things sorted: `2026-04-05-title.md`
- The `description` field is what shows on the card preview on the website
- You can use headers (`##`), bullet lists (`-`), bold (`**text**`), and tables in your content
- Don't edit files in the `docs/` folder or any `.html` files — those are auto-generated
